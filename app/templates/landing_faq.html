FAQ

Answers for teams deciding whether BidBoard can actually hold up during rush week.

These are the questions chapter leadership usually asks before moving off spreadsheets: fit, access control, mobile execution, rollout, and whether the data stays durable in production.

Fit + Setup

Who is BidBoard built for?

BidBoard is built for fraternity and sorority recruitment teams that need structured, collaborative decision-making across heads, officers, and members.

Can multiple organizations use one deployment?

Yes. Each organization has a separate tenant path, isolated database, custom branding, and independent approvals.

How hard is rollout?

Most teams start with the interactive demo, configure one organization workspace, then move rush officers into live use once branding and storage are set.

Security + Access

How does access control work?

Role-based visibility is enforced at the API layer. Head roles, officers, and members see different scopes, and users cannot edit or delete other users’ ratings.

Can organizations customize branding?

Yes. Logos, colors, organization titles, and workspace identity can be customized independently per tenant.

Is accountability preserved?

Yes. Leadership can see rating changes, deltas, and comments where allowed, while member-facing views remain intentionally limited.

Mobile + Workflow

Does BidBoard work on mobile during events?

Yes. Mobile-specific pages are optimized for small screens so officers can search, rate, assign, and follow up from iPhone or Android.

Can we import rushees from forms?

Yes. You can collect submissions through Google Forms, export CSV responses, and import them into BidBoard with validation and duplicate handling.

Can the team coordinate in one place?

Yes. Calendar events, lunches, goals, notifications, assignments, and officer chat are part of the product workflow rather than side systems.

Data Durability

How are backups handled?

BidBoard supports CSV export bundles, SQLite snapshots, and season archive/reset tools for cyclical recruitment operations.

Will data disappear when we deploy new code?

Not when persistent storage is configured correctly. BidBoard includes startup checks and admin diagnostics to verify that the configured paths are durable.

Can we keep historical context between seasons?

Yes. Archived season data can be preserved while the live board resets for the next rush cycle.

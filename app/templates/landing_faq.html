Frequently Asked Questions

Answers for teams evaluating BidBoard.

Everything below is built to help chapter leaders move fast without sacrificing fairness, security, or accountability.

Who is BidBoard built for?

BidBoard is built for fraternity and sorority recruitment teams that need structured, collaborative decision-making across officers and members.

Can multiple organizations use one deployment?

Yes. Each organization has a separate tenant path, isolated database, branding controls, and independent account approvals.

How does access control work?

Role-based visibility is enforced at the API layer. Head roles, officers, and members see different data scopes based on configured permissions.

Does BidBoard work on mobile during events?

Yes. Mobile-specific pages and routing are included so team members can update ratings, notes, and lunches from iPhone or Android.

Can we import rushees from forms?

Yes. You can collect submissions via Google Forms, export CSV responses, and import directly into BidBoard with validation and duplicate checks.

How are backups handled?

BidBoard supports CSV export bundles and SQLite snapshots. It also includes season archive + reset tooling for cyclical recruitment operations.

Will data disappear when we deploy new code?

Not when persistent storage is configured correctly. BidBoard includes startup checks and admin diagnostics to verify that storage paths are durable.

Can we customize branding per organization?

Yes. You can upload logos and set organization-level themes so each chapter workspace reflects its own identity.
